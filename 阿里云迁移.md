## 二、迁移目标架构

```
用户 → 阿里云 CDN/DCDN → 阿里云 ECS (Next.js SSR) → Supabase (数据库, 不变)
                                        → Clerk (认证, 不变)
                                        → 通义千问 (AI, 不变)
```

**关键决策：**
- 数据库继续用 Supabase（不需要迁移）
- 认证继续用 Clerk（不需要迁移）
- AI 接口继续用通义千问（不需要迁移）
- 仅将 **Next.js 应用服务器** 迁移到阿里云 ECS

## 三、执行流程

### Phase 1: 代码整理与提交（当前优先）

### Phase 2: 购买阿里云资源

#### 2.1 购买 ECS 服务器

| 项目 | 推荐配置 | 说明 |
|------|----------|------|
| 地域 | 华东（杭州）或华北（北京） | 离用户近，延迟低 |
| 实例规格 | ecs.c7.large（2核4G） | Next.js 基础够用，可随时升配 |
| 系统 | Ubuntu 22.04 LTS | 官方长期支持 |
| 存储 | 40GB SSD 高效云盘 | 代码 + node_modules + 日志 |
| 带宽 | 按量付费 或 固定 3-5Mbps | 前期按量更省钱 |
| 数量 | 1 台 | - |

#### 2.2 购买域名

- 在阿里云「域名服务」购买域名（如 `promptgenius.cn` 或 `promptgenius.com`）
- 完成 **实名认证**（企业或个体均可）
- 如果用户在中国大陆境内，需要 **ICP 备案**（通过阿里云代备案系统提交，约 7-20 个工作日）

#### 2.3 域名解析

购买完成后，在阿里云「云解析 DNS」添加：
- `A 记录`：`@` → ECS 公网 IP
- `A 记录`：`www` → ECS 公网 IP

### Phase 3: ECS 服务器环境搭建

SSH 登录 ECS 后执行：

```bash
# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 安装 Node.js 20.x（与生产环境一致）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 3. 验证
node -v   # 应输出 v20.x
npm -v    # 应输出 10.x

# 4. 安装 PM2（进程管理）
sudo npm install -g pm2

# 5. 安装 Nginx（反向代理）
sudo apt install -y nginx

# 6. 安装 Git
sudo apt install -y git
```

### Phase 4: 部署应用

```bash
# 1. 创建应用目录
mkdir -p /var/www/prompt-genius
cd /var/www/prompt-genius

# 2. 拉取代码
git clone https://github.com/barronli0523/prompt-genius.git .

# 3. 安装依赖
npm install --production

# 4. 创建 .env 文件
cat > .env << 'EOF'
# 从本地 .env.local 复制所有变量
NEXT_PUBLIC_SUPABASE_URL=https://wzgtllbonthwrvbwvggw.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=...
CLERK_SECRET_KEY=...
OPENAI_API_KEY=...
EOF

# 5. 构建
npm run build

# 6. 用 PM2 启动
pm2 start npm --name "prompt-genius" -- start -- -p 3000
pm2 save
pm2 startup   # 设置开机自启
```

### Phase 5: Nginx 反向代理 + HTTPS

```bash
# 1. 配置 Nginx
sudo tee /etc/nginx/sites-available/prompt-genius << 'EOF'
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
EOF

# 2. 启用站点
sudo ln -s /etc/nginx/sites-available/prompt-genius /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx

# 3. 安装 Certbot 并申请 HTTPS 证书
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

# 4. 设置证书自动续期
sudo systemctl enable certbot.timer
```

### Phase 6: Clerk 配置更新

在 [Clerk Dashboard](https://dashboard.clerk.com) 中：
1. 进入 **Application → Configure → General**
2. 在 **Allowed redirect URLs** 中添加：
   - `https://your-domain.com/**`
3. 在 **Allowed CORS Origins** 中添加：
   - `https://your-domain.com`
4. 将环境变量从 test 系列切换为 production 系列（如需要）

### Phase 7: 数据库迁移

## 五、安全注意事项

1. **密钥管理**：`.env` 文件绝不提交到 Git，已通过 `.gitignore` 排除
2. **防火墙**：ECS 安全组仅开放 80、443、22 端口
3. **HTTPS**：必须配置，Clerk 和生产环境强制要求
4. **ICP 备案**：中国大陆服务器绑定域名必须备案，否则 80/443 端口会被封
5. **Supabase 密钥**：区分 anon key（前端用）和 service_role key（仅后端用）

## 七、备用方案

如果不想自己运维 ECS，可以考虑：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **阿里云函数计算 (FC)** | 无需运维，按量付费 | 冷启动延迟，不适合长连接 |
| **阿里云容器服务 ACK** | 可扩展 | 配置复杂，成本高 |
| **Vercel 商业版** | 零运维 | 国内访问可能不稳定 |
| **Cloudflare Pages** | 免费 + CDN | 国内访问依赖 CF 节点，有不确定性 |

**推荐**：前期用 ECS 最直接，可控性强，成本可控。用户量上来后再考虑容器化。
